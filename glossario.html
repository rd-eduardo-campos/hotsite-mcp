Glossário de MCP e IA, em português

Definições curtas e diretas dos termos que aparecem quando o assunto é MCP, IA e integração. Sem tecniquês — para você ler qualquer documentação ou conversa técnica sem se perder.

Os essenciais

Os 5 termos que você precisa saber antes de qualquer coisa

Se você entender estes cinco, consegue acompanhar qualquer conversa sobre MCP — mesmo entre desenvolvedores.

MCP sigla
Sigla para Model Context Protocol, em português "protocolo de contexto para modelos de IA". É o padrão criado para que IAs (Claude, ChatGPT, etc.) consigam acessar dados e executar ações em sistemas externos de forma segura e estruturada.
Servidor MCP infra
O sistema que fica do lado do produto — no nosso caso, da RD Station — esperando os pedidos da IA. Quando você "ativa o MCP da RD", está ligando o servidor MCP da RD à sua IA. A RD Station tem três servidores: um para CRM, um para Marketing e um para Conversas.
Cliente MCP infra
O programa que a IA usa para conversar com o servidor MCP. Quando você usa o Claude Desktop com MCP, o Claude Desktop é o cliente MCP. Outros exemplos: n8n, Make, ChatGPT (em rollout), Cursor.
Tool capacidade
Cada ação que a IA pode executar através do MCP é uma tool (ferramenta). Criar um contato é uma tool, listar negociações é outra, inscrever em fluxo é outra. O MCP da RD Station tem 94 tools no total.
Agente de IA conceito
Uma IA que age — não só responde. Quando a IA usa tools do MCP para executar coisas (criar, atualizar, mover), ela está agindo como agente. É a diferença entre um assistente que sugere e um operador que faz.
IA em geral

Termos do universo da inteligência artificial

Modelo de IA conceito
O "cérebro" da IA. Claude, GPT-4, Gemini e Llama são exemplos de modelos. Cada um foi treinado de forma diferente e tem habilidades específicas.
Prompt uso
É o pedido em texto que você manda para a IA. Pode ser uma pergunta simples ("quantos leads entraram hoje?") ou uma instrução longa com regras.
Token técnico
Unidade de medida da IA — equivale, em média, a 4 letras de texto. As IAs cobram por token processado. Pense em "tokens" como "minutos de fala" da IA.
Contexto conceito
Tudo o que a IA "lembra" durante uma conversa. Inclui a pergunta original, as respostas anteriores e qualquer dado que foi puxado via MCP. Cada IA tem um limite de contexto (medido em tokens).
API técnico
Sigla para "Application Programming Interface". É a forma "tradicional" de sistemas conversarem — exige código, chave de acesso e documentação técnica. O MCP é a evolução: faz o mesmo trabalho, mas em linguagem natural.
LLM sigla
Sigla para "Large Language Model" — em português, "modelo grande de linguagem". É o nome técnico para o tipo de IA que você usa no dia a dia (Claude, ChatGPT, etc.).
Generativa adjetivo
IA "generativa" é a que cria conteúdo novo — texto, imagem, código. Diferente de IA "preditiva", que prevê coisas (lead vai converter ou não), ou "classificatória" (esse email é spam ou não).
Agentic / agêntico adjetivo
IA "agêntica" é a que tem capacidade de agir, não só conversar. Ela toma decisões, executa passos e usa ferramentas (tools) para chegar ao objetivo. O MCP é o que torna a IA agêntica dentro da RD Station.
Detalhes do MCP

Termos técnicos que aparecem em conversas mais avançadas

Tool call técnico
O momento exato em que a IA decide usar uma tool. Por exemplo: você pergunta "quantos deals abertos?", a IA reconhece e faz um "tool call" para a ferramenta deals_list.
Resource conceito
É um pedaço de dado que o servidor MCP pode expor para a IA — diferente de uma tool (que executa ação). Por enquanto, a RD Station expõe principalmente tools, não resources.
Prompt template avançado
Modelos de prompt que o servidor MCP pode oferecer para a IA usar. Por exemplo, um template "analise as 10 negociações mais paradas" que o usuário só precisa selecionar. Recurso opcional do MCP.
OAuth segurança
Padrão de autorização usado pelo MCP para você dar permissão à IA. É o mesmo fluxo de "Entrar com Google" que você usa em outros apps — você aprova, não compartilha senha.
stdio / SSE / WebSocket técnico
São formas diferentes de cliente e servidor MCP se comunicarem. Você não precisa escolher: a documentação do cliente que você usa (Claude Desktop, n8n) já configura sozinho.
Schema técnico
É a "ficha técnica" de cada tool: o nome, o que ela faz, quais parâmetros aceita. A IA lê os schemas para decidir qual tool usar e como chamá-la.

Termos próprios da RD que você vai encontrar

Copiloto de IA do RD Station Marketing. Cria emails, sugere listas, otimiza envios. É a IA nativa do Marketing — funciona em conjunto com o MCP, não no lugar dele.
Rê produto
Copiloto de IA do RD Station CRM. Atualiza negociações por voz, prioriza leads quentes, sugere próxima ação. É a IA nativa do CRM.

Não confunda

Pares de termos que parecem iguais, mas não são

Automação ≠ IA

Automação segue regras fixas: "se A acontecer, faça B". IA entende contexto e decide com julgamento. Um fluxo de RD Marketing é automação. A LYNN escolhendo o melhor horário de envio é IA.

Chatbot ≠ Agente de IA

Chatbot tradicional segue script: "digite 1 para X". Agente de IA entende linguagem natural, acessa sistemas e age. O MCP transforma uma IA em agente real.

API ≠ MCP

Os dois conectam sistemas. Mas a API exige código, autenticação técnica e documentação. O MCP é uma camada por cima: o usuário fala em texto, a IA chama as APIs por baixo.

Servidor ≠ Cliente

O servidor MCP fica do lado do produto (RD Station). O cliente MCP fica do lado da IA (Claude Desktop, n8n). Servidor "responde", cliente "pergunta".

Modelo ≠ Aplicativo

Modelo é o cérebro (GPT-4, Claude 4.7, Gemini). Aplicativo é a interface (ChatGPT, Claude Desktop, Gemini app). Um modelo pode estar em vários apps, e um app pode usar vários modelos.

Tool ≠ Plugin

No padrão MCP, tool é cada ação disponível (criar, listar, atualizar). Plugin era o termo antigo do ChatGPT para integrações. MCP substitui o conceito de plugin com mais segurança e portabilidade.
