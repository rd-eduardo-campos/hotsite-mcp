Skills em construção pela RD Station

Esta biblioteca documenta as skills que a RD usa internamente nas operações com o MCP. O botão "Baixar .md" ainda não está ativo — quando os arquivos estiverem prontos, você poderá baixar a definição e usar com a sua IA (Claude Skills, ChatGPT Custom GPTs, agentes em n8n, etc.). Por enquanto, este catálogo serve como blueprint do que é possível combinar com as ferramentas do MCP.

🧭 Gestor 👑 Admin CRM

Auditoria de Pipeline Semanal

Diagnóstico do funil de vendas pronto para a reunião semanal: distribuição por etapa, tempo médio, gargalo identificado e top motivos de perda do período.

Ferramentas MCP usadas
funnel_list funnel_stages_list deals_list lost_reasons_list
Passos da skill
  1. Lista os funis de vendas ativos
  2. Para o funil escolhido, lista todas as etapas
  3. Para cada etapa, lista as negociações abertas
  4. Calcula tempo médio de permanência por etapa
  5. Lista motivos de perda do período
  6. Devolve diagnóstico com gargalo e top 3 deals em risco
single-product · 6 passos

Triagem de Negociações Frias

Identifica deals abertos sem atividade há mais de X dias, cria tarefas de follow-up para o responsável e registra nota explicando a ação. Reduz vendas que esfriam por esquecimento.

Ferramentas MCP usadas
deals_list users_list tasks_create deals_notes_create
Passos da skill
  1. Lista deals abertos sem atividade há mais de [X] dias
  2. Para cada deal, identifica o responsável atual
  3. Cria tarefa de follow-up vinculada (vencimento: amanhã)
  4. Adiciona nota no deal documentando a ação
  5. Devolve resumo do que foi feito por vendedor
single-product · 5 passos

Auditoria Completa de Fluxo

Diagnostica a saúde de um fluxo de automação específico: configuração, quem entrou, quem chegou em cada ação, quem saiu. Aponta onde está o maior abandono.

Ferramentas MCP usadas
workflows_search workflow_get_details workflow_leads_started_list workflow_leads_at_action_list workflow_leads_exited_list
Passos da skill
  1. Busca o fluxo pelo nome
  2. Recupera configuração completa (ações, status, trigger)
  3. Lista quem entrou no período
  4. Para cada ação, lista quem chegou ali
  5. Lista quem saiu (concluiu ou abandonou)
  6. Calcula taxa de progressão e identifica abandono
single-product · 6 passos
🛠️ Operador 🧭 Gestor Marketing
